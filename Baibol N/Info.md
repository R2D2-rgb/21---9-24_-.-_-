## Роль: Технический писатель / QA (Аналитик) — «Мозг и Контролер»
## Основные обязанности:

Анализ требований: Внимательное изучение технической документации, бизнес-спецификаций и пользовательских сценариев для понимания функциональности системы.
Разработка документации:
Создание подробных технических описаний архитектуры, API, схем баз данных.
Подготовка руководств пользователя, инструкций по эксплуатации и документации для разработчиков.
Обновление и поддержка актуальности документации по мере развития проекта.

Планирование тестирования:
Разработка тест-кейсов, сценариев и чек-листов для проверки функциональности системы.
Формирование тестовой документации для автоматизации и ручного тестирования.

Контроль качества:
Проведение тестирования программного обеспечения.
Анализ и фиксация ошибок, несоответствий.
Взаимодействие с командой разработки для устранения дефектов.

Обеспечение соответствия:
Проверка соответствия реализованных функций требованиям и спецификациям.
Валидация данных и бизнес-логики.

Коммуникация:
Взаимодействие с разработчиками, менеджерами и заказчиками для уточнения требований.
Обеспечение прозрачности и точности информации по проекту.
## Цель:
Обеспечить высокое качество, точность и полноту технической документации и программного продукта посредством системного анализа требований, разработки тестовых сценариев и контроля соответствия реализованных функций заданным спецификациям, тем самым способствуя надежной и стабильной работе проекта.
